# AgentDock 隐私声明（草案）与数据收集清单（T0.9-23）

上架华为应用市场所需材料（设计文档 §9.6 / §26.5）。本文两部分：（一）面向用户的
隐私声明草案；（二）逐项数据收集清单。所有条目均以仓库内实现为依据，标注出处。

---

## 一、隐私声明草案

### 1. 默认原则：本地优先，数据不出端

AgentDock 是本地优先的智能体应用。默认情况下，您的所有数据——会话、知识库、
智能体配置、记忆——只存储在本设备的应用沙箱内（RDB 加密数据库与沙箱文件区），
**不上传到任何服务器**。断网状态下本地模型、本地知识库、本地记忆完整可用
（设计文档 §2、§9）。

### 2. 哪些数据在什么条件下会离开设备

仅以下两类，且都以您的显式操作为前提：

1. **远程模型调用**：当您显式添加并选用远程模型服务（如 OpenAI 兼容 API）时，
   当前对话内容，以及被您授权为「可出端」（`allow_remote`）的知识库中被
   检索到的引用片段，会发送给您自己选定的服务商。知识库默认为
   「仅本地」（`local_only`），此类内容由隐私围栏在数据装配层**强制过滤**，
   不进入任何远程请求——过滤发生在代码的数据层，不依赖界面约定（设计文档
   §9.2，Schema `privacy_level` 字段）。此外，随远程调用一同发送的还有您为该
   服务商配置的 API Key（仅作为鉴权凭据发给对应服务商本身）。
   蜂窝网络下默认禁用远程模型（「仅 Wi-Fi 使用远程模型」出厂开启，§3.3）。
2. **您显式导出的备份文件**：备份包（.adbk）由您主动导出到自选位置，此后由您
   自行保管与传输。备份包**默认剔除所有密钥**（红线 16）；因跨设备还原需要，
   包内数据为明文 JSONL，请妥善保管。

除此之外没有第三种数据出端路径。

### 3. 崩溃记录：仅本机，零上传，默认关闭

崩溃/冻屏记录开关出厂默认**关闭**。开启后仅在本机日志记录系统捕获的崩溃摘要
（异常名/入口/时间戳等代码信息，不含会话与文档内容——红线 20），供您日后自行
导出诊断包排查问题。**不存在任何网络上报后端**（`CrashWatcher.ets`、
`SettingsViewModel.ets`）。

### 4. API Key 的存储

远程服务的 API Key 仅存入系统 Asset Store Kit（系统密钥库，硬件级保护），
绝不写入数据库、配置文件或备份包（§9.1；数据库 providers 表只存密钥别名，
没有密钥列）。

### 5. 我们不做的事

- **无账号体系**：应用无需注册登录，不采集任何身份信息。
- **无遥测**：不内置任何统计/分析/埋点上报。
- **无广告 SDK**：不集成任何广告或第三方追踪组件。
- **权限最小化**：仅申请网络权限与经系统文件选择器授权的文件读取；
  不申请定位、通讯录等无关权限（§9.3）。

## 二、数据收集清单

| 数据类型 | 存储位置 | 是否离开设备 | 用户控制方式 |
|---|---|---|---|
| 会话（消息、对话记录） | 本机 RDB 加密数据库（沙箱内） | 仅当用户选用远程模型时，当前对话内容发送给用户自选的服务商；本地模型下不出端 | 选用本地模型；删除会话；「仅 Wi-Fi 使用远程模型」开关 |
| 知识库文档（原文与切块） | 沙箱文件区（原文）+ 本机 RDB（切块与索引） | 默认否（`local_only`）；仅当用户将库设为「可出端」且选用远程模型时，被检索到的引用片段随请求发送 | 库级隐私级别开关（默认仅本地）；删除文档/整库 |
| 智能体配置（名称、提示词、工具授权等） | 本机 RDB | 否 | 应用内编辑、删除 |
| 模型文件（本地模型权重） | 沙箱文件区（运行时由用户触发下载） | 否（仅下载入端，不上传） | 应用内下载、删除 |
| API Key | 系统 Asset Store Kit（密钥库），不落数据库/配置文件/备份包 | 仅作为鉴权凭据随用户显式发起的远程调用发送给对应服务商 | 应用内录入、删除；不配置则不存在 |
| 日志与崩溃记录 | 本机环形缓冲 + 系统 hilog；不落用户内容明文与密钥 | 否（零上传，无上报后端） | 崩溃记录开关（默认关闭） |
| 备份包（.adbk） | 用户导出时写入用户自选位置 | 仅在用户显式导出后以文件形式离开沙箱，由用户自行保管 | 用户主动导出/导入；包内默认不含密钥 |
