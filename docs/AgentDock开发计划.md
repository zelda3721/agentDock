# AgentDock 开发计划

**依据**：《鸿蒙智能体平台系统设计》v1.0–v1.4（`docs/鸿蒙智能体平台系统设计.md`，997 行）
**计划版本**：v1.0 · 2026-07
**排期基线**：§26.4 三版本切分（V0.9 / V1.0 / V1.5 / V2）；§12/§20/§24.4 的 M0–M8 仅作工作量与验收标准参考
**资源假设**：个人开发（1 人全栈。WBS 与周表中 A=原生/内核向、B=ArkTS 应用向仅为工作性质泳道标签，均由本人承担；法务/PM 类事项由本人自办，必要时外部咨询）
**排期粒度**：周。W1 = 项目启动周；周序表达依赖顺序与相对工作量，进度按实际滚动，**不做容量核算**；**V0.9 上架为 W8 末目标节点（§26.7 窗口期考量）**

---

## 1. 概述

### 1.1 项目目标

在 HarmonyOS 6.0+（API 20）上交付鸿蒙三端（手机/平板/2in1）第一个"模型自选、数据在端、记忆可审计"的开源智能体工作台（§26.2）。核心闭环：接入模型（本地 GGUF / 远程 OpenAI 兼容）→ 挂载本地知识库 → 定义并运行智能体 → 沉淀记忆 → 定期自动整理反哺对话（§1）。

### 1.2 设计文档基线

| 文档版本 | 内容 | 对本计划的效力 |
|---|---|---|
| v1.0 | §0–§14 主体架构、M0–M5 | 架构与工作项来源；排期数字仅作参考 |
| v1.1 | §15–§20 语音/多模态、M6/M7 | V1.5 工作项来源 |
| v1.2 | §21–§24 修订 R1–R5、开源合规、上下文治理、Long Job | R1–R5 为强制修订，逐条吸收 |
| v1.3 | §25–§26 PDF 分层管线、PM 复审与路线图重切 | §26.4 为**权威路线图**；§25.3 OCR/VL 归 V2 |
| v1.4 | §27 长程压缩六机制 M1–M6 + 回归门禁 | 按归属功能拆入各版本（见 §2.4） |

### 1.3 适用范围

本计划覆盖 V0.9 / V1.0 / V1.5 三个可发布版本的周级排期（共 20 周）与 V2 深水区 backlog（不排周），含 WBS、验收标准、发布检查单、测试/合规/风险管理。不覆盖设计文档"非目标"项（端侧微调、多用户体系、跨端分布式推理，§1）。

## 2. 总体策略

### 2.1 三个可发布版本 + V2（§26.4）

按"先占位 → 再差异化 → 后亮点"组织，反对 24 周憋全功能版本：

- **V0.9 占位版（W1–W8）**：抢结构性窗口期（竞品移植窗口估 12–24 个月，§26.3），尽早上架优先；
- **V1.0 差异化版（W9–W14）**：记忆全套 + 工具总线，验证"记忆/知识库用户留存更高"假设（§26.5）；
- **V1.5 亮点版（W15–W20）**：语音电话模式 + 两级大脑，制造传播素材；
- **V2 深水区**：由 V1.x 留存数据决定优先级，禁止提前建设（§26.4）。

### 2.2 风险前置验证

最大不确定项排在最前面的周（§12 原则）：

- **W1–W2**：llama.cpp OHOS 交叉编译与三真机性能 spike（全项目最大技术风险，W2 末必须出结论）；
- **W3**：中文分词双路线 spike（含 RDB 能否加载 SQLite 扩展的 API 验证），W3 末定案；
- **W1 启动**：生成式 AI 备案边界法务评估（发布前置项，§26.7）与商标注册（防套壳，§26.7）。

### 2.3 每版发布判据（一票否决）

| 版本 | 发布判据 |
|---|---|
| V0.9 | 上架应用市场 + GitHub/Gitee 开源仓库公开 + 激活漏斗埋点跑通（§26.4）；备案法务结论已取得（§26.7） |
| V1.0 | 记忆功能上线并通过 M4 级验收（强杀续跑、7 天无膨胀）；留存假设埋点上线（§26.4/§26.5） |
| V1.5 | 断网电话模式达延迟指标（首响 ≤1.8s/≤1.2s、打断 ≤400ms）；传播素材产出（§26.4/§20-M6） |
| V2 各项 | 满足 §8 节所列数据驱动启动条件后方可立项 |

### 2.4 v1.4 §27 机制的版本归属原则

§27 六机制按其作用对象所在版本排入，不整体后置：

- **V0.9**（随 L1/L3 交付）：M1 溯源指针纪律（摘要段/外置桩必带 step_range/artifact 指针）、M2 保真门之**机械实体核对**（正则：`\d{2,}`/混合串/路径/URL）、§27.4 出厂参数（水位 70%/85%/50%、近窗 K=6、anchor ≤25%）；
- **V1.0**（随 L2/工具总线/Agent-as-Tool 交付）：M1 之 `artifact.read`/`trace.read` 工具、M6 子代理 512/512 合同、M3 目标复诵（Interactive 版复诵节 ≤200 token）、压缩回归套件 v1 入 CI；
- **V2**（随 Long Job 交付）：M2 完整约束寄存器（Nano 抽取入 TaskLedger anchor）、M3 每 10 步自检、M4 事实带证据入册、M5 段边界对齐、§27.3 全量金标（50+ 步、约束召回 100%）。

### 2.5 排期与缓冲策略

- 总排期：8（V0.9）+ 6（V1.0）+ 6（V1.5）= 20 周，为依赖顺序与相对工作量的参考骨架；个人开发不做容量核算，进度按实际滚动，版本边界与范围划分保持不变。
- 缓冲以三种形式内置：
  1. **范围缓冲**：每版 MoSCoW 的 Should/Could 项即裁剪缓冲，进度吃紧时按各版预定义的裁剪顺序收缩，Must 项不动；
  2. **spike 产物复用**：S1 产物直接演进为 llama_bridge 骨架、S2 产物直接进检索流水线，避免重复投入；
  3. **发布周纪律**：W8/W14/W20 为封版周，只做验收、修复与发布材料，不排新功能。
- V1.0/V1.5 期间兼顾上一版线上问题响应；若问题量大，按裁剪顺序砍 Should 项，优先保 Must。

## 3. 硬约束与工程红线清单

开发全程不可违反，Code Review 与 CI 按此执行：

**平台约束（§0）**
1. 全原生 ArkTS + C++/NAPI，禁止任何 Android 生态库复用（§0-1）；目标 HarmonyOS 6.0+/API 20，DevEco Studio 6.x。
2. 单 HAP 三 deviceTypes + 三层工程（common/feature/product）+ 断点布局覆盖三端，禁止为 PC 单独立项（§0-2/ADR-5）；PC 窗口/键鼠/快捷键在 product 层注入。
3. 本地推理主引擎 = llama.cpp（GGUF，OHOS NDK aarch64 交叉编译 + NAPI）；MNN 仅备选插槽（§0-3/ADR-1）。
4. 生产容量按纯 CPU 设计；NPU 只是预留插槽，禁止任何里程碑依赖 NPU（§0-4）。
5. RAG/Agent 内核自建；HMAF/系统 RAG/小艺只做适配层，禁止形成依赖（§0-5/ADR-3）。
6. 重后台任务（记忆整理等）必须可中断、可断点续跑，挂 workScheduler + 充电/空闲/电量>40% 条件，禁止假设常驻后台（§0-6/ADR-4）。
7. 原生层统一 C/C++（musl/TLS 坑），CI 含真机 .so 加载冒烟（§0-7）。

**架构与修订红线（§2/§21）**
8. 向量存储 = RDB(SQLite) 元数据 + FTS5 + 原生 vec_index(hnswlib)，禁止独立向量数据库进程（ADR-2）。
9. Agent 运行时自研（受控 ReAct + JSON 工具协议，≤2k 行），禁止 LangChain 类框架（ADR-3）。
10. **R1**：PDF 解析强制 PDFium（BSD-3）；MuPDF（AGPL）永久禁用，许可扫描 CI 防再引入（§21-R1/§22.4）。
11. **R2**：ContextBuilder 装配顺序 = [稳定前缀] system+Agent+L3 画像 → [半稳定] 会话历史 → [易变尾部] 本轮记忆+RAG+用户消息；禁止易变内容置于历史之前（§21-R2）。
12. **R3**：本地推理队列优先级抢占：voice > interactive chat > 整理/摘要；抢占 = AbortSignal，被抢任务幂等可重入（§21-R3）。
13. **R4**：Agent 循环必须带循环检测（工具签名重复 ≥2）与停滞检测（3 步相似度 >0.95），先"换思路"再终止出部分结果（§21-R4）。
14. **R5**：仓库与安装包一律不内置模型权重；运行时下载 + 清单（含许可/出处）下载前展示（§21-R5）。

**PDF 管线（§25）**
15. 逐页路由：文本页 PDFium 直取（V0.9）；OCR 档/VL 档整体归 V2，禁止提前占用 V0.9/V1.x 排期（§25.3/§26.4）；PDFium 不可移除（§25.1）；OCR/VL 产物标 provenance 并 UI 提示可跳原图（§25.3）。

**安全隐私（§9）**
16. API Key 只存 Asset Store Kit，禁止落 RDB/配置/日志；备份默认剔除密钥（§9.1/§3.3）。
17. 隐私围栏强制在 ContextBuilder 数据层执行（local_only 内容对 remote Provider 过滤 + UI 提示），禁止靠 UI 逻辑；语音远程链路与远程 VL 同受此拦截（§9.2/§17-B/§25.3）。
18. 权限最小化：仅 INTERNET + filePicker 授权文件读；无定位/通讯录/全盘（§9.3）。
19. 禁止任意 shell/代码执行工具；http.fetch 域名白名单 + 每 Agent 首次确认（§9.4）。
20. 日志不落用户明文与密钥；崩溃收集默认关闭（§9.5）。

**开源合规（§22）**
21. 本项目 Apache-2.0；依赖白名单 MIT/BSD/Apache/ISC/Zlib/PD；GPL/AGPL/LGPL/SSPL/专有 CI 即 fail（§22.1/§22.4）。
22. 仓库自 V0.9 公开起必备：LICENSE/NOTICE/THIRD_PARTY_LICENSES/models manifest/CONTRIBUTING(DCO)/全源文件 SPDX 头（§22.4）。
23. 华为 SDK 二进制与签名材料（.p7b/.cer/keystore）禁止入仓（§22.2）。
24. SenseVoice 等语音模型许可 V1.5 前逐款人工核对；TTS 音色只收可商用者入官方清单（§22.2）。

**长程压缩纪律（§27，按 §2.4 归属版本）**
25. 压缩只作用于工作集，run_steps/run_artifacts 永远保留全量原始轨迹；摘要/折叠桩必带溯源指针（M1）。
26. L3/L4 摘要必过保真门（实体机械核对起步），不过门换保守档重压（M2）。
27. 压缩逻辑改动没有回归套件不许合并（§27.3，**无豁免期**）：合并门禁自压缩逻辑首次交付（V0.9 W8，随 T0.9-17/T0.9-22 交付 V0.9 最小子集：L1 外置溯源指针可寻回 + L3 摘要实体机械核对 + 植入约束召回）起生效；套件随 V1.0 W13（T1.0-12）扩展为 v1 全量（含 L2 折叠与 trace.read 补救核对）。
28. §27.4 出厂参数为默认值（可配）：anchor ≤25%、水位 70%/85%/50%、近窗 K=6/3、复诵 ≤200 token、子代理合同 512/512、trace.read ≤2k token。
